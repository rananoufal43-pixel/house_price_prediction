LAB MANUAL
Dockerizing the House Price ML Application
🎯 Aim

To containerize a Flask-based Machine Learning application using Docker and run it successfully inside a Docker container.

🧰 Software Requirements

Windows 10 / 11 (64-bit)

Python installed

Flask project ready

Docker Desktop installed

WSL2 enabled

📁 Project Structure

Before starting Docker, ensure your project folder looks like this:

house_price_project/
│
├── app.py
├── requirements.txt
├── Dockerfile
├── model/
│   └── house_price_model.pkl
├── templates/
│   └── index.html
└── static/
    └── style.css
🧱 STEP 1: Create Dockerfile

Inside the main project folder:

Create a file named:

Dockerfile

⚠️ Important:

File name must be exactly Dockerfile

No .txt

No extension

Must be inside main folder

📝 STEP 2: Add Content to Dockerfile

Paste the following code:

# Use official Python image
FROM python:3.10

# Set working directory
WORKDIR /app

# Copy project files
COPY . .

# Install dependencies
RUN pip install --no-cache-dir -r requirements.txt

# Expose Flask port
EXPOSE 5000

# Start application
CMD ["python", "app.py"]
🔄 STEP 3: Update app.py (Very Important)

Change this line:

app.run(debug=True)

TO:

app.run(host='0.0.0.0', port=5000)
Why?

Docker needs 0.0.0.0
Otherwise, the browser cannot access the app.

🏗 STEP 4: Build Docker Image

Open terminal inside project folder.

Run:

docker build -t house-price-app .
Explanation:

build → create image

-t house-price-app → name of image

. → current folder

First build may take 2–3 minutes.

If successful, you will see:

Successfully built
Successfully tagged house-price-app
▶ STEP 5: Run Docker Container

Now run:

docker run -p 5000:5000 house-price-app
Explanation:

Left 5000 → your PC

Right 5000 → Docker container

house-price-app → image name

🌐 STEP 6: Open in Browser

Open:

http://localhost:5000

🎉 Your Dockerized ML App is running!

🧠 What Happened?

You:

✔ Created Docker image (blueprint)
✔ Started Docker container (running app)
✔ Opened app from browser
✔ ML prediction works
✔ Application is now portable

🏗 System Architecture
Browser
   ↓
Docker Container
   ↓
Flask App
   ↓
Saved ML Model
   ↓
Prediction
⚠️ Common Errors and Solutions
❌ Error 1: Docker Not Recognized
'docker' is not recognized
Reason:

Docker not installed.

Solution:

Download Docker Desktop from:
https://www.docker.com/products/docker-desktop/

Install it.

Restart system.

Run:

docker --version
❌ Error 2: WSL Not Installed

If Docker shows WSL error:

Run in PowerShell (Admin):

wsl --install

Restart PC.

❌ Error 3: Port Already Allocated
Bind for 0.0.0.0:5000 failed
Reason:

Another container is running.

Solution:

Check running containers:

docker ps

Stop container:

docker stop <container_id>
📌 Important Points

Dockerfile must not have extension.

Flask must run on 0.0.0.0.

Port mapping is necessary.

Docker must be installed separately.

Always build before running.





















LAB MANUAL – NEXT PROCEDURE
After Running Docker Container Successfully
🎯 Aim of This Section

To:

Run container in background

Manage Docker containers

Push image to DockerHub

Deploy the Docker image to a free cloud platform (Render)

🔹 PART 1: Running Container in Background
▶ STEP 1: Run in Detached Mode

Instead of blocking terminal, run:

docker run -d -p 5000:5000 house-price-app
Explanation:

-d → Run container in background

Terminal remains free

▶ STEP 2: Check Running Containers
docker ps

Output shows:

Container ID

Image name

Running status

Port mapping

▶ STEP 3: Stop Container
docker stop <container_id>

Example:

docker stop 123abc456
▶ STEP 4: Remove Container (Optional Clean Up)
docker rm <container_id>

This removes stopped container.

📌 Learning Point

Image = Blueprint
Container = Running Instance

You can create multiple containers from one image.

🔹 PART 2: Push Docker Image to DockerHub
🎯 Aim

To upload Docker image to public registry for cloud deployment.

▶ STEP 1: Tag Image
docker tag house-price-app yourusername/house-price-app

Example:

docker tag house-price-app venkateshmoodu/house-price-app
▶ STEP 2: Login to DockerHub
docker login

Enter:

Username

Password

If successful:

Login Succeeded
▶ STEP 3: Push Image
docker push venkateshmoodu/house-price-app

Image will upload to DockerHub.

📌 Verification

Visit:

https://hub.docker.com

Check repository → Image should appear.

🔹 PART 3: Deploy Docker Image to Render (Free Cloud)
🎯 Aim

To deploy Docker image publicly using free cloud hosting.

▶ STEP 1: Login to Render

Visit:

https://render.com

Sign up using Gmail.

▶ STEP 2: Create Web Service

Click:

New → Web Service

Choose:

Deploy from Docker image
▶ STEP 3: Enter Image URL

Enter:

docker.io/venkateshmoodu/house-price-app:latest
▶ STEP 4: Configure Settings

Service Name:

house-price-app

Instance Type:

Free

Port:

5000

Add Environment Variable:

Key:

PORT

Value:

5000
▶ STEP 5: Deploy

Click:

Deploy Web Service

Wait 2–4 minutes.

▶ STEP 6: Access Public URL

Render will generate URL like:

https://house-price-app.onrender.com

Open it.

🎉 Application is now live globally.
